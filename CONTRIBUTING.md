# Contributing to VideoCap / 贡献指南

Thank you for your interest in contributing to VideoCap! / 感谢您对 VideoCap 的贡献兴趣！

## How to Contribute / 如何贡献

### Report Bugs / 报告 Bug

Before creating bug reports, please check existing issues. / 创建 Bug 报告前，请先检查现有 Issue。

**When submitting a bug report, include:** / 提交 Bug 时请包含：

- A clear and descriptive title / 清晰描述性的标题
- Steps to reproduce / 复现步骤
- Expected behavior / 预期行为
- Actual behavior / 实际行为
- Screenshots if applicable / 相关截图
- Environment info (OS, Node.js version) / 环境信息（操作系统、Node.js 版本）

### Suggest Features / 功能建议

- Explain the use case / 说明使用场景
- Describe the expected behavior / 描述预期行为
- Consider alternatives / 考虑替代方案

### Pull Requests / 提交代码

1. Fork the repository / Fork 仓库
2. Create a feature branch (`git checkout -b feature/amazing-feature`) / 创建功能分支
3. Commit your changes (`git commit -m 'Add amazing feature'`) / 提交更改
4. Push to the branch (`git push origin feature/amazing-feature`) / 推送到分支
5. Open a Pull Request / 创建 Pull Request

## Development Setup / 开发环境

```bash
# Install dependencies / 安装依赖
npm install

# Run in development mode / 开发模式运行
npm start
```

## Code Style / 代码风格

- Follow existing code conventions / 遵循现有代码规范
- Use meaningful variable names / 使用有意义的变量名
- Add comments for complex logic / 为复杂逻辑添加注释

## Commit Messages / 提交信息

Use clear and descriptive commit messages: / 使用清晰描述性的提交信息：

```
feat: add new feature
fix: resolve bug
docs: update documentation
style: format code
refactor: restructure code
```
